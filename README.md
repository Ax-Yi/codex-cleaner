# Windows 本地电脑清理助手

## 一键打包脚本

项目根目录提供 `build_exe.ps1`，用于一键生成 Windows 单文件 exe。

在项目根目录打开 PowerShell，执行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\build_exe.ps1
```

脚本会自动完成：

* 清理旧的 `build/`、`dist/` 和 `*.spec`
* 创建 `.venv` 虚拟环境
* 安装 `requirements.txt`
* 使用 PyInstaller 打包
* 输出 `dist\WindowsCleanerAssistant.exe`

脚本使用的打包方式等价于：

```powershell
pyinstaller --onefile --noconsole --name WindowsCleanerAssistant run.py
```

脚本不会使用 `--add-data`，因此不会把扫描报告、缓存、`.idea`、`__pycache__` 或临时文件作为项目资源打进 exe。

一个面向新手的 Windows 本地电脑清理工具，使用 Python 和 Tkinter 编写。

项目默认只扫描和生成报告，不会自动删除文件。删除操作需要用户手动选择，并进行二次确认，适合用于电脑空间整理、Python 项目缓存清理、重复文件检查和隐私风险文件名排查。

## 功能特性

* 扫描大文件
* 扫描重复文件
* 扫描 Python 项目缓存
* 扫描疑似隐私文件名
* 支持暂停扫描、继续扫描和停止扫描
* 提供常用扫描预设，快速选择下载、桌面、文档、视频、图片、临时目录和 NVIDIA 缓存
* 清理报告顶部自动生成清理建议汇总
* 自动保存扫描历史记录，可在界面中查看和清空
* 在已扫描结果中搜索路径或文件名
* 扫描结果支持勾选多选，不需要按 Ctrl 或 Shift
* 支持全选、取消全选、只选择推荐清理
* 清理按钮为“移动勾选项到回收站”，只处理已勾选项目
* 为每条结果显示风险等级和处理建议
* 按风险等级筛选结果，支持只显示推荐清理
* 表格支持按大小从大到小排序
* Excel 支持导出全部结果或勾选结果
* Excel 导出包含“清理建议”工作表
* 支持重复文件分组查看、推荐保留和只勾选重复副本

* 默认只扫描，不会主动删除任何文件

## 项目背景

日常使用 Windows 电脑时，下载目录、桌面、Python 项目缓存、临时文件和重复文件容易不断堆积。

普通清理工具可能直接删除文件，存在误删重要资料的风险。本项目的目标是做一个更安全、更透明的本地电脑清理助手。

程序会先扫描文件并生成结果列表，用户可以查看文件路径、大小、修改时间和风险类型，再决定是否删除。

## 主要功能说明

### 1. 大文件扫描

扫描指定目录中的大文件，帮助用户快速发现占用空间较多的文件。

适合检查：

* 下载目录中的安装包
* 桌面上的压缩包
* 视频素材
* 长期未整理的大型文件

### 2. 重复文件扫描

通过文件大小和哈希值判断重复文件，减少误判。

适合检查：

* 重复下载的安装包
* 多次保存的压缩包
* 重复图片或文档
* 重复项目备份

### 3. Python 项目缓存清理

识别 Python 项目中常见的缓存文件和临时目录。

可识别内容包括：

* `__pycache__`
* `.pyc`
* `.pytest_cache`
* `build`
* `dist`
* 临时日志文件
* 空文件夹

### 4. 隐私文件名扫描

扫描疑似包含隐私信息的文件名，但不会读取或上传文件内容。

可识别关键词包括：

* `.env`
* `cookie`
* `cookies`
* `token`
* `api_key`
* `apikey`
* `secret`
* `password`
* `passwd`
* `authorization`

这个功能适合在上传 GitHub 前检查项目，降低误传敏感文件的风险。

### 5. 扫描控制和结果搜索

扫描任务运行在后台线程，不会卡住 Tkinter 界面。

扫描控制按钮包括：

* 开始扫描
* 暂停扫描
* 继续扫描
* 停止扫描

暂停扫描会保留已经扫描到的结果；继续扫描会从暂停位置继续；停止扫描会保留当前结果，并允许重新选择目录后重新开始扫描。

扫描结果页提供“扫描结果搜索”，只在已扫描结果中搜索路径或文件名，不会重新扫描磁盘，也不会删除文件。

搜索、风险等级筛选、只显示推荐清理和大小排序都只影响当前显示结果，不影响原始扫描结果，也不会清空已勾选状态。

### 6. 常用扫描预设

扫描目录输入框下方提供“常用扫描预设”按钮，可以快速填入常见扫描目录：

* 下载目录：当前用户 `Downloads`
* 桌面：当前用户 `Desktop`
* 文档：当前用户 `Documents`
* 视频：当前用户 `Videos`
* 图片：当前用户 `Pictures`
* Python 项目目录：优先尝试 `D:\py xiangmu`，不存在时弹出目录选择窗口
* 系统临时目录：`tempfile.gettempdir()`
* NVIDIA 缓存：`C:\Users\当前用户\AppData\Local\NVIDIA\DXCache`

如果预设目录不存在，程序会提示“该目录不存在，请手动选择目录。”

默认扫描目录是当前用户下载目录；如果下载目录不存在，则使用用户主目录。程序不建议直接扫描整个 C 盘，用户手动输入 `C:\` 或选择整个 C 盘后，开始扫描前会弹窗确认。

### 7. 勾选、多选和批量处理

扫描结果表格最左侧提供“选择”列：

* `☐` 表示未勾选
* `☑` 表示已勾选
* 点击“选择”列可以在勾选和取消之间切换
* 可以直接勾选多行，不需要按 Ctrl 或 Shift

表格上方提供三个批量按钮：

* 全选：勾选当前表格中显示的结果
* 取消全选：取消当前表格中显示的结果
* 只选择推荐清理：只勾选风险等级为“推荐清理”的结果

路径搜索、风险等级筛选和大小排序不会清空已勾选状态。“移动勾选项到回收站”只处理已勾选的项目；风险等级为“不建议删除”的项目即使被勾选，也会被拦截。

移动前会进行二次确认，并显示：

* 勾选数量
* 可处理数量
* 被安全策略拦截数量
* 预计释放空间

移动到回收站依赖 `send2trash`。如果没有安装，程序会提示运行：

```powershell
python -m pip install send2trash
```

Excel 导出时可以选择导出全部结果或只导出勾选结果。

### 8. 重复文件分组处理

扫描到重复文件后，可以点击“查看重复文件分组”打开重复文件分组窗口。

窗口会按 `DUP-0001` 这类重复组编号展示：

* 组内文件数量
* 每个文件的完整路径
* 文件大小
* 修改时间
* 风险等级
* 处理建议

每个重复组会自动生成“推荐保留文件”。推荐规则是优先保留修改时间最新的文件；如果修改时间相同，则保留路径更短的文件。包含 `Windows`、`Program Files`、`ProgramData`、`AppData\Roaming\Tencent`、`WeChat`、`WXWork`、`QQ` 等路径时，程序会提示“请手动确认”，避免自动选择高风险文件。

重复文件分组窗口提供：

* 只勾选重复副本：每组保留推荐文件，只勾选同组内其余“推荐清理”副本
* 清空重复文件勾选：取消重复文件相关勾选
* 复制重复组信息：把重复组路径、大小、修改时间、推荐保留和勾选状态复制到剪贴板

重复文件分组窗口中的勾选状态会同步回主表格。移动到回收站时仍然只处理已勾选项目，风险等级为“不建议删除”的项目会继续被安全策略拦截。

Excel 导出包含“重复文件分组”工作表，记录重复组编号、文件路径、大小、修改时间、是否推荐保留、是否被勾选、风险等级和处理建议。

### 9. 风险等级和处理建议

每条扫描结果都会显示风险等级和处理建议：

* 推荐清理：常见缓存、日志、转储等路径，例如 `Temp`、`DXCache`、`GraphicsCache`、`__pycache__`、`.pyc`、`.log`、`.dmp`
* 谨慎处理：常用软件数据路径，例如 `Tencent`、`WeChat`、`WXWork`、`QQ`、`JetBrains`、`Microsoft\Edge`、`CapCut`
* 不建议删除：系统或程序关键文件，例如 `.dll`、`.exe`、`.sys`、`Windows`、`Program Files`、`ProgramData`、`pagefile.sys`

“不建议删除”的项目默认禁止通过界面删除。

### 10. 清理建议汇总

扫描完成后，“清理报告”区域顶部会自动显示“清理建议汇总”，包括：

* 总结果数量和总文件大小
* 推荐清理、谨慎处理、不建议删除的数量和总大小
* 重复文件组数量
* 已勾选数量和已勾选预计释放空间

程序会根据路径生成提示，例如 Python 缓存、Temp、NVIDIA DXCache、GraphicsCache 可以优先关注；Tencent、WeChat、WXWork、QQ 目录需谨慎处理；`pagefile.sys`、`Windows`、`Program Files` 不建议处理。

清理建议只做提示，不会自动勾选，也不会自动处理文件。

### 11. 报告导出

Excel 导出包含“清理建议”工作表，内容包括扫描目录、扫描时间、风险等级统计和建议文字。

### 12. 扫描历史记录

每次扫描正常完成后，程序会在本地 `history.json` 中保存一条扫描历史记录。

历史记录包括：

* 扫描时间
* 扫描目录
* 结果数量
* 总大小
* 推荐清理数量和大小
* 谨慎处理数量和大小
* 不建议删除数量和大小
* 重复文件组数量

主界面提供“查看扫描历史”按钮，可以打开历史记录列表，也可以清空历史记录。

`history.json` 已加入 `.gitignore`，不会上传到 GitHub。

### 12. 扫描历史记录

每次扫描正常完成后，程序会在本地 `history.json` 中保存一条扫描历史记录。

历史记录包括：

* 扫描时间
* 扫描目录
* 结果数量
* 总大小
* 推荐清理数量和大小
* 谨慎处理数量和大小
* 不建议删除数量和大小
* 重复文件组数量

点击“查看扫描历史”可以打开历史记录窗口。窗口中可以查看历史列表，也可以清空所有历史记录。

`history.json` 只保存在本地，已经加入 `.gitignore`，不会上传到 GitHub。

扫描完成后，可以将结果导出为 Excel 文件，方便查看和归档。

报告内容包括：

* 文件路径
* 文件大小
* 修改时间
* 文件类型
* 风险提示
* 建议处理方式

## 安全策略

本项目以安全为优先目标，默认不会自动删除文件。

安全机制包括：

* 默认行为是扫描和生成报告
* 删除前需要用户手动选择文件
* 删除前会弹出确认窗口
* 删除确认需要输入指定文字
* 风险等级为“不建议删除”的项目默认禁止删除
* 自动跳过系统目录
* 自动跳过无权限访问的目录
* 隐私扫描只检查文件名，不读取文件内容
* 不会上传任何本地文件
* 不会连接远程服务器

默认跳过的目录包括：

```text
C:\Windows
C:\Program Files
C:\Program Files (x86)
C:\ProgramData
$Recycle.Bin
System Volume Information
```

## 使用建议

第一次使用时，建议只扫描，不删除。

推荐流程：

```text
1. 选择需要扫描的目录
2. 执行扫描
3. 查看扫描结果
4. 导出报告
5. 手动判断是否需要删除
6. 再执行删除操作
```

不建议直接扫描整个 C 盘。更推荐从下面这些目录开始：

```text
下载
桌面
文档
视频
Python 项目目录
```
